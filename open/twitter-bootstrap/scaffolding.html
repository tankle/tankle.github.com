全局样式 预设样式，排版，链接

需要HTML5文档类型

Bootstrap利用了HTML5元素和CSS性，需要HTML5文档类型。确保在你的页面开始包含了HTML5文档类型。

<!DOCTYPE html>
<html lang="en">
  ...
</html>

排版和链接

在基建文件<strong>scaffolding.less</strong>中，我们设置了基本的全局显示、排版和链接样式，特别的：

  • 移除body中的margin属性
  • 在body上设置background-color: white;
  • 使用@baseFontFamily, @baseFontSize，和@baseLineHeight属性作为字体排版的基本样式
  • 通过@linkColor设置全局的链接样式，只在hover上应用下划线

通过Normalize框架重置样式

在Bootstrap 2.0版本中，传统的CSS reset（为去除浏览器间的差异而自定义的CSS代码段）逐渐利用Normalize.css的元素，Normalize.css由Nicolas Gallagher创建，是一个基于HTML5 Boilerplate的项目。

新的重置（reset）任然可以在reset.less中找到，但是为了简洁和准确起见很多元素已经被删除。

用百分比代替像素数

流动网格布局系统使用百分比代替固定的像素数来表示列宽，它 和固定式的网格布局一样，也采用了响应式的设计，用来确保适应不同的屏幕分辨率和设备。

流动行

要想让固定行流动起来，只需要简单的将.row改为.row-fluid。列保持不变，简单的操作就可以在固定和流动之间随意切换。

例如

<div class="row-fluid">
  <div class="span4">...</div>
  <div class="span8">...</div>
</div>

流动嵌套

流动网格嵌套有一点不同：嵌套的列数不需要匹配外层的列，由于每一行都100%的占满了外层的列，所有的列在每一级都会被重置。

定制网格

变量 默认值 描述
@gridColumns 12 列数
@gridColumnWidth 60px 列宽
@gridGutterWidth 20px 列距

LESS中的变量

创建Bootstrap只用了很少一部分变量，只是用来定制上边提到过的默认940px的网格布局，所有的变量都保存在variables.less。

如何定制

定制网格意味着你需要改变三个@grid*变量，并且重新编译Bootstrap。改变variables.less中的网格变量，可以用四种重新编译的方法之一来实现。如果你想增加更多的列，要确保在grid.less中增加相应的CSS文件。

保持响应式

只有默认的940px宽的固定网格布局的定制才需要考虑响应式。为了保证Bootstrap响应式的特性，你需要改变responsive.less中的布局属性。

布局设计 用基本的模板创建网页

固定布局

使用默认的940px宽居中的布局方式，只需要在文件中包含<div class="container">。

<body>
  <div class="container">
    ...
  </div>
</body>

流动布局

<div class="container-fluid">提供了更灵活的网页结构、最小最大宽度以及左侧边栏，用来做web应用或文档不错。

<div class="container-fluid">
  <div class="row-fluid">
    <div class="span2">
      <!--Sidebar content-->
    </div>
    <div class="span10">
      <!--Body content-->
    </div>
  </div>
</div>

响应式设计 不同设备和屏幕分辨率的media queries

Media queries能做什么

设备类型（Media queries）允许基于比例（ratios）、宽度（widths）和显示类型（display type）等来定制CSS，但是通常集中于min-width和max-width。

  • 修改网格中的列宽
  • 必要的地方使用层叠元素（stack elements）代替float元素
  • 调整标题和文本的字体大小用以适应更多的设备

使用media queries是可靠的，但是对你的移动用户来说仅能作为一个开始。对于更大的工程，是需要专用的代码而不是只依靠media queries。

支持的设备

Bootstrap只支持一小部分设备类型来帮助你的项目适应更多的设备和屏幕分辨率：

类型 布局宽度 列宽 列距
智能手机（Smartphones） 480px及以下 流动列宽，无固定值
智能手机到平板（Smartphones to tablets） 767px及以下 流动列宽，无固定值
肖像平板（Portrait tablets） 768px及以下 42px 20px
默认（Default） 980px及以上 60px 20px
大屏幕（Large display） 1200px及以上 70px 30px

需要加入meta标签

为了确保设备显示合适的响应式页面文件头中需要包含viewport meta标签。

<meta name="viewport" content="width=device-width, initial-scale=1.0">

使用media queries

Bootstrap不会自动的包含media queries，但是理解和加入media queries是很简单的，只需要简单的配置。你可以选择不同的方式使用Bootstrap的响应式（responsive）特性：

  1. 使用编译过的版本bootstrap-responsive.css
  2. 增加@import "responsive.less"，重新编译Bootstrap
  3. 作为一个独立的文件改变和重新编译responsive.less

为什么不是仅仅的包含进去？ 老实说，不是任何地方都需要响应式（responsive）。当然也不鼓励开发者移除这个特性，我们最好的配置方法是：

  // Landscape phones and down
  @media (max-width: 480px) { ... }

  // Landscape phone to portrait tablet
  @media (max-width: 767px) { ... }

  // Portrait tablet to landscape and desktop
  @media (min-width: 768px) and (max-width: 979px) { ... }

  // Large desktop
  @media (min-width: 1200px) { ... }
